5 Ways to become the MacGyver of Go
Getting productive the Standard Library

5 Ways to become the MacGyver of Go, with nothing more than a paperclip, an elastic band, a magnet and the Go Standard Library

Install the Go toolset from source.

This might sound crazy, as there are package installers available for most operating systems. But it has one really useful side effect. You can then create a cross compilation toolchain. So you can easily release multiple versions of your Go app. for instructions, click here.

One great tool for this is “mitchellh/gox”.

$ go get github.com/mitchellh/gox
...
$ gox -h
...

Before you use Gox, you must build the cross-compilation toolchain. Gox can do this for you. This only has to be done once (or whenever you update Go):

$ gox -build-toolchain
...

Once that is done, you’re ready to cross compile!

Pimp your favourite editor

I don’t care how long you have been programming, nobody can remember the entire standard libray for a language.

This tends to create bad habits, as you can end up putting great effort into writing a piece of code and then find out later that it already exists in the standard library.

THios is where code completion, syntax highlighters come in to there own, as they can speed up the familiarity with the SL.

For Go, there are a number of great ones, but in my opinion, GoSublime for SublimeText2 / 3 is above them all, as it works with the installed version, so update your go toolchain and the code completion gos with it. It will also complete for your own packages.

  • GoSublime, link

Learn the Standard Library

Save yourself the pain of reinventing the wheel and become familiar with the standard Library. Some good resources for this are the following books:

  • Go, The Standard Library, Daniel Huckstep
  • The Way to Go: A Thorough Introduction to the Go Programming Language, by Ivo Balbaert

Make use of http://godoc.org

GoDoc is a Go package search engine that hosts documentation for Go packages on Bitbucket, GitHub, Launchpad and Google Project Hosting.

It should be the first stop when creating a project that requires a new, complex package to be written, as it is quite likely that the functionlity already exists.
