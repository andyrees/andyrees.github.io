About
welcome to my blog

Hi I’m Andy and welcome to my blog.

I live in Devon in the UK with my wife Rebecca and enjoy working out, photography, programming and technology in general. I like programming but I love taking mundane tasks and simplifying them with code.

I started programming with BASIC in the 70s on a BBC Micro, Assembly code at University, then C and C++ later, Bash, Java & C#. At University I studied Electronic Engineering BENG (HONS), but came away with skills in business, advanced engineering mathematics, Digital & analogue electronics, programming.

From Python to Go

Like the writer of the XKCD blog, Python allowed me to enjoy programming again; once you get used to the white space or get a decent ide like Pycharm. But my recent obsession is Golang.

Python might be nice to write, but it can be complex to deploy. However, Go has inferred runtime static typing, compiles to an executable with no runtime dependencies and is garbage collected and is as much a pleasure to code as python.

Go is increasingly the answer for both small and large projects, in my day to day activities. My recent adventures with broadcast regulations and perceived loudness, ended with quite a nice example of Go’s effectiveness resulting in a really useful utility.

Give it a R.E.S.T.

I recently wrote the Ooyala client API in GO, which was fun and enabled me to run a lot of imports and updates from an old system very quickly.

The most fun I have had so far is to write a REST based transcode engine in go that leverages FFMPEG and inotify on Ubuntu. The result is a prioitisable, opensource transcode engine with a web front end in angularjs.

Go makes this so easy and with the REST backend, I could easily create a massive transcode nextwork and control it all via a master REST controller.
